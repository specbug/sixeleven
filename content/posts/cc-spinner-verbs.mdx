Claude Code has quietly become my favorite way to work with AI. Not because of any single feature, but because it's designed to get out of your way and let you customize the parts that matter to you. Here's how I've set mine up.

## Terminal: Ghostty + Zellij

I run Claude Code inside [Ghostty](https://ghostty.org/) with [Zellij](https://zellij.dev/) as a multiplexer. This combo gives me:

- **Performance** - Ghostty is GPU-accelerated and noticeably snappier than iTerm2 or Terminal.app, especially when Claude is streaming long outputs
- **Parallel Claude sessions** - I run multiple Claude Code instances side by side, each working on different tasks or repos. `Alt+n` spawns a new pane, `Alt+h/l` to jump between them
- **Session persistence** - Zellij serializes sessions to disk, so I can detach and reattach without losing context. Useful when switching between projects

My Ghostty config (`~/.config/ghostty/config`):

```text
font-family = "R Plex Mono"
font-size = 19.0
font-thicken = true
theme = ayu
macos-option-as-alt = true
copy-on-select = true
scrollback-limit = 50000
cursor-style = bar
cursor-style-blink = false
```

The key bits: `macos-option-as-alt` lets Alt-based shortcuts work properly (Zellij uses these heavily), and a large scrollback buffer helps when reviewing Claude's longer outputs.

Zellij config (`~/.config/zellij/config.kdl`) is mostly default with vim-style navigation and tmux compatibility mode (`Ctrl+b`). The important additions:

```kdl
session_serialization true
serialize_pane_viewport true
scrollback_lines_to_serialize 5000
```

This saves session state including pane contents, so I can close my laptop and pick up exactly where I left off.

## Sound Notifications

This one's simple but transformative. When Claude finishes a task or needs my attention, I want to know—without staring at the terminal.

```json
"hooks": {
  "Stop": [
    {
      "matcher": "",
      "hooks": [
        {
          "type": "command",
          "command": "afplay /System/Library/Sounds/Hero.aiff"
        }
      ]
    }
  ],
  "Notification": [
    {
      "matcher": "",
      "hooks": [
        {
          "type": "command",
          "command": "afplay /System/Library/Sounds/Submarine.aiff"
        }
      ]
    }
  ]
}
```

`Stop` fires when Claude completes work—Hero.aiff is a satisfying triumphant chime. `Notification` fires when Claude pauses for input or wants your attention—Submarine is a gentle "hey, look over here." macOS has a bunch of built-in sounds in `/System/Library/Sounds/` to choose from.

## Status Line

The default status line is fine, but I wanted something closer to my shell prompt. This shows user@host, the current directory, git branch (if any), and active Python virtualenv:

```json
"statusLine": {
  "type": "command",
  "command": "d=$(basename \"$(pwd)\"); g=\"\"; if git rev-parse --git-dir >/dev/null 2>&1; then b=$(git symbolic-ref --short HEAD 2>/dev/null || git rev-parse --short HEAD 2>/dev/null); if [ -n \"$b\" ]; then g=$(printf \" \\033[36m(%s)\\033[0m\" \"$b\"); fi; fi; v=\"\"; if [ -n \"$VIRTUAL_ENV\" ]; then v=$(printf \" \\033[32m(%s)\\033[0m\" \"$(basename \"$VIRTUAL_ENV\")\"); elif [ -n \"$CONDA_DEFAULT_ENV\" ]; then v=$(printf \" \\033[32m(%s)\\033[0m\" \"$CONDA_DEFAULT_ENV\"); fi; printf \"\\033[2m%s@%s\\033[0m \\033[34m%s\\033[0m%s%s\" \"$(whoami)\" \"$(hostname -s)\" \"$d\" \"$g\" \"$v\""
}
```

It's a bit of a one-liner, but it gives you something like `rishitv@mac my-project (main) (venv)` at a glance.

## Plugins

I've enabled a few community plugins:

- **commit-commands** - shortcuts for git workflows
- **pr-review-toolkit** - code review helpers
- **agent-sdk-dev** - for building Claude Agent SDK apps
- **ralph-loop** - autonomous coding loops

```json
"enabledPlugins": {
  "pr-review-toolkit@claude-code-plugins": true,
  "agent-sdk-dev@claude-code-plugins": true,
  "commit-commands@claude-code-plugins": true,
  "ralph-loop@claude-plugins-official": true
}
```

## Other Settings

A few more things I keep enabled:

```json
"model": "opus",
"alwaysThinkingEnabled": true
```

Opus is slower but noticeably better for complex reasoning tasks. Extended thinking (`alwaysThinkingEnabled`) lets Claude work through problems step by step before responding—makes a real difference for debugging and architecture decisions.

---

All of this lives in `~/.claude/settings.json`. The [Claude Code docs](https://docs.anthropic.com/en/docs/claude-code) cover the full schema, but honestly the best way to figure out what works for you is just to experiment.
